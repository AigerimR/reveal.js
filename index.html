Material UI

Hello, my name is Aigerim and today I’m going to talk about Material UI. What kind of library it is and how you can use it in your project. So basically, when you’re creating an application sometimes it is quite useful to use some external libraries that will help you to keep the design of your project consistent and at the same time decrease the amount time spend on development and stylization. So MUI is an option for developers to use with React applications.

Google

Material Design

Material UI

It is based on Material design, which is basically a set of rules and guidelines created by Google for their own websites and applications. However, later some developers took those standards and implemented them in Material UI. And created a set of ready to use React components, that is basically an HTML elements or set of elements that have a build-in styles and animations in it.

Install


							npm install @mui/material @emotion/react @emotion/styled
							
							yarn add @mui/material @emotion/react @emotion/styled
It is quite straightforward to use. You just have to install npm packages as your dependencies and then you’re ready to go.

and....use

So, here I’m going to show you the use of simple components that are often used, which are Button, List, Input field, You import them as components from Material UI library and use it as any other React component in your file. On the right hand side you can see that those components already have some predefined styles, because MUI already assigned to them some of the properties.
And library actually provides quite a good list of components that could start from a simple button component and go on to more complex components like set of buttons, that already are given as pagination component, navbar component. What is more, it actually have also icon component.

icons

So basically for icons MUI have an extra library. So you use it to extra an icon you need, import it in your file and use it again as any other React component. You can see icons on the right hand side as well.

typography

One more interesting component that MUI has is called Typography. It basically, a standardized style for the text in the library. You have to wrap up the text inside of the typography component. And set up an element that you want this component to look like. Here I’m showing h3 component with the use of Typography stylization and just h3 element by itself. And you can see the difference on the right hand side. But however, you may be want at some point to change a little bit of those standardized styles in your project. Or may be just change the whole style that is already predefined and that could be done by several options that I’m going to describe and explain further on in this presentation. Because, generally MUI is already having some predefined classes for the components. So it is not advisable to change the class names as you usually do in CSS or React components. So there are other ways

customization

standard properties & sx
First of them is using the standardized properties that are already assigned to each component in the library. Like each component, could it be a button or input, already have such properties as variant/color/size for example. And each of these properties have set of values that could be changed as well. For example in the color you can change secondary to primary or error color. You can choose it from there. But if by any chance you won’t find properties that you want from this list, you can also use sx property, which receives an object with properties that you want to change and basically assigns them to the component you want. But as you see these are sort of instyle change of styles. That’s why if you’re going to use these kind of options, you might need to use it every instance of the component that you want to choose. Let’s say, if you have several buttons, then you will need to change each button like that. On the right hand side you can see how button changed and how margin have already added up to it.

customization

extended sx
So to overcome this issue you can use the same sx property but with a constant instead. And the constant could be assigned to object of properties. Basically I look at it as if this option is actually replicating classname assignment to the component as if you do it in css when you give a class to the element. You basically can use this constant and sx property and assign those to all different buttons that you want to change.

customization

Theme
If at some point you actually would want to change the whole file or a part of it. You can also use a thing called theme and ThemeProvider. ThemeProvider is basically a Component that you use to wrap up the part of the code that you want to be affected. And you assign a theme property to it with an argument that you create form CreateTheme method. To it you assign a different kind of values and properties that you want to change. Here accordingly I changed the primary color and a standardized h3 typography and created my own typography style as well. As you see I have already created a fontFamily and the link to these you have to include yourself in HTML file. On the right hand side this is the theme I have changed for the whole file and I wrapped up into typography h3 element here and h3 over here. That’s why they have changed and the rest have been left as it was.

customization

create new component
If by any chance, you want to not change the whole theme or just take a components or set of components. You can use another way to stylize it. And this is creation of a new component. This is achieved with the help of styled utility that’s you have to install as well form the library. And with its help you take the standardized component and change the properties you want in it, creating a new component by yourself which you export and then use as a normal component in your react file. on the right hand side you can see the button that was created using this new component.
